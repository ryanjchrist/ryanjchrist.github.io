16-bit MIPS CPU Design

Computer Architecture Digital Logic Logisim

April 2025

Context

As part of Duke University's Computer Architecture course, I designed and implemented a 16-bit MIPS-like RISC CPU using Logisim. This project provided hands-on experience in CPU design principles, instruction set architecture, and digital logic implementation. The design focuses on a word-addressed architecture that incorporates basic CPU components and a single-cycle execution model.

Design & Approach

The project involved creating a functional 16-bit RISC architecture based on the MIPS instruction set. Built entirely in Logisim, this design offers a visual approach to understanding how processors execute instructions, manage data, and handle operations. The implementation focuses on educational clarity while maintaining architectural correctness.

Technical Implementation

Architecture Overview

The CPU follows a 16-bit RISC (Reduced Instruction Set Computing) architecture, designed for simplicity and educational clarity. The design is word-addressed, meaning each memory location corresponds to a 16-bit word, which simplifies instruction handling and data management.

Key Components

The implementation includes all fundamental CPU components:

  • Instruction Decoder: Efficiently decodes different instruction types and generates control signals
  • Arithmetic Logic Unit (ALU): Performs arithmetic and logical operations
  • Register File: Manages CPU registers for data storage and retrieval
  • Program Counter: Tracks instruction execution flow
  • Memory Management: Handles instruction fetching and data memory operations

Instruction Set

The CPU supports a subset of MIPS instructions, including arithmetic operations (ADD, SUB), logical operations (AND, OR), memory operations (LOAD, STORE), and control flow instructions (BRANCH, JUMP). All instructions execute in a single clock cycle, simplifying the design and making it easier to understand the execution pipeline.

Single-Cycle Design

The project focuses on implementing a single-cycle CPU, meaning every instruction completes in one clock cycle. This design choice eliminates the complexity of pipelining while still demonstrating core CPU concepts like instruction fetching, decoding, execution, and memory access.

Results & Outcomes

The completed design successfully executes MIPS instructions and demonstrates the fundamental principles of CPU architecture. Through Logisim's visual simulation capabilities, I was able to trace instruction execution, observe data flow through components, and validate the correctness of the design. This project provided deep insights into how processors work at the hardware level.

Key Achievements

  • Designed and implemented a functional 16-bit MIPS-like RISC CPU
  • Created efficient instruction decoding unit for multiple instruction types
  • Implemented complete data path with ALU, registers, and memory management
  • Validated design through Logisim simulation and testing
  • Gained comprehensive understanding of CPU architecture and digital logic

Challenges & Solutions

One of the main challenges was designing an efficient instruction decoder that could handle different instruction formats while generating appropriate control signals. I solved this by creating a modular decoding unit that extracts instruction fields and uses them to generate control logic. Another challenge was ensuring proper data flow through the CPU components, which required careful attention to signal timing and component interconnections.

Key Learnings

  • CPU Design Principles: Deep understanding of instruction sets, data handling, and control logic
  • Digital Logic: Practical experience with how ALUs, registers, and memory work together in a CPU
  • Logisim Proficiency: Mastered using Logisim for visualizing and simulating complex digital circuits
  • RISC Architecture: Comprehensive knowledge of RISC architectures and their design principles
  • System Integration: Understanding how individual components integrate to form a complete CPU
